```sh
cd my-app
pnpm install
pnpm dev
```

Uzumaki is a desktop UI runtime for people who like React's component model but do not want to ship a browser.

You write TypeScript and JSX. Uzumaki creates native windows, renders a small set of native elements, and exposes low-level runtime APIs for paths, windows, events, clipboard, and custom renderer work.

## A Tiny App

```tsx
import { useState } from 'react';
import { Window } from 'uzumaki';
import { render } from 'uzumaki-react';

const window = new Window('main', {
  width: 900,
  height: 620,
  title: 'Counter',
  rootStyles: { bg: '#0b0b0f', color: '#f4f4f5' },
});

function App() {
  const [count, setCount] = useState(0);

  return (
    <view display="flex" flexDir="col" items="center" justify="center" h="full" gap={18}>
      <text fontSize={32} fontWeight={700}>
        {count}
      </text>
      <button
        px={18}
        py={10}
        rounded={10}
        bg="#f59e0b"
        color="#111827"
        hover:bg="#fbbf24"
        onClick={() => setCount((value) => value + 1)}
      >
        <text fontWeight={700}>Count up</text>
      </button>
    </view>
  );
}

render(window, <App />);
```

## What Makes It Different?

Uzumaki is closer to a framework runtime than a browser shell:

- `Window` creates real native windows.
- JSX elements like `<view>`, `<text>`, and `<button>` are Uzumaki elements, not HTML tags.
- Layout and style props are explicit runtime props, not CSS files.
- The `uzumaki` module is provided by the runtime, so app bundles mark it external.
- React is the first renderer, and the low-level runtime API is available for future renderers.
